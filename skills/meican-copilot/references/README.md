# meican-copilot Knowledge Base

This repository is the long-term work copilot knowledge base. It is organized for two goals:
1. Fast routing: find the right playbook in 1-3 jumps.
2. Deep execution: every critical topic has detailed procedures, anchors, and scripts.

This knowledge base intentionally uses standard Markdown only (no Obsidian-specific syntax).

## Start Here
- `INDEX.md`: scenario-based navigation.
- `CONTRIBUTING.md`: writing and update rules.
- `CHANGELOG.md`: inbox and weekly consolidation log.

## Domains
- `architecture/`: platform topology, runtime model, service catalog.
- `development/`: coding standards, proto/gateway conventions, planning patterns, examples.
- `operations/`: incident workflows, log/trace/db playbooks, failure patterns.
- `infra/`: ArgoCD/Terraform/CI/CD/observability/network access controls.
- `capabilities/`: app capabilities, dependency maps, scenario index.
- `principles/`: macro principles, decision frameworks, risk checklists.
- `templates/`: fast-start templates for app/incident/design.
- `legacy/`: detailed historical references, to be migrated and removed per checklist.
