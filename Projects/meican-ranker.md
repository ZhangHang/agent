## 设计要点（来源：design.md）

### 系统与数据流

- 在线排序服务仅做 Redis 读 + 公式计算，不负责召回或复杂训练。
- CDC 消费写入 Doris 事实表，日聚合 + 滚动窗口后同步 Redis。
- 配置通过 Nacos 下发，启动加载，不做热更新。

### 数据与存储

- 三层结构：fact → daily aggregate → window(7/30/90d)。
- 滚动窗口按日增量更新，窗口不包含当天数据。
- Redis 使用 MGET 读取特征，identity 维度加 TTL 控制规模。

### 排序逻辑

- 公式打分（变量来自各维度特征），缺失维度按 0 处理。
- 新商家置顶：窗口期 + 数量限制 + 创建时间排序。

### Job 调度

- 仅每日批处理：01:00 聚合 → 01:05 window → 01:10 sync。
