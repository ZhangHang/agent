# 油罐车轨迹报告优化

## 增加指标的算术平均值

```markdown
# Add Average Calculation for Temperature, Pressure, and Concentration

## Backend Changes (`internal/service/truck_dashboard.go`)

1. **Update** `**CompartmentStatsJSON**` **struct** (line 643):

- Add `AvgP float64` for average pressure
- Add `AvgT float64` for average temperature
- Add `AvgC float64` for average concentration
- Include JSON tags: `json:"avg_p"`, `json:"avg_t"`, `json:"avg_c"`

2. **Modify** `**buildStatsAndSeriesForWindow**` **function** (line 1005):

- For compartments: Track `sumP`, `countP`, `sumT`, `countT` while iterating through points
- For operation boxes: Track `sumC`, `countC` while iterating through points
- After the loop, calculate averages: `avgP = sumP / countP`, `avgT = sumT / countT`, `avgC = sumC / countC`
- Set the `AvgP`, `AvgT`, `AvgC` fields in the stats maps before returning

## Frontend Changes (`internal/net/http/handler/static/truck_report.html`)

1. **Update compartment stats display** (around line 499-503):

- Change "压力 (最小/最大)" to "压力 (最小/平均/最大)"
- Update display from `{{ sdata.min_p.toFixed(1) }} / {{ sdata.max_p.toFixed(1) }}` to include average
- Similarly update temperature display
- Update the "no data" fallback section (line 512-516) to show three values

2. **Update operation box stats display** (around line 526-527):

- Change "浓度 (最小/最大)" to "浓度 (最小/平均/最大)"
- Update display from `{{ sdata.min_c.toFixed(2) }} / {{ sdata.max_c.toFixed(2) }}` to include average
- Update the "no data" fallback section (line 536-537)

## Implementation Details

- Track sum and count during point iteration in `buildStatsAndSeriesForWindow`
- Calculate averages only when count > 0 to avoid division by zero
- Use same decimal precision: 1 decimal for pressure/temperature, 2 decimals for concentration
- Handle edge case: if no data points, average should be 0 or undefined (check if count > 0)
```


### 解决 GPS 偏移

```markdown
Add Location Merge Radius Override for GPS Jitter Handling

## Description

### Problem

Some destination locations experience GPS jitter, causing multiple separate stop/movement segments when trucks are parked at the same location, leading to fragmented tracks and inaccurate statistics.

### Solution

Implemented a post-processing merge step that consolidates segments within a configurable merge radius for specific locations:

- Added Override struct with MergeRadiusMeters field to Location

- Created mergeSegmentsForOverrideLocations function that:

- Merges consecutive stops within radius (30min time window)

- Detects and removes GPS jitter movements (both endpoints within radius)

- Preserves real movements (endpoint outside radius)

- Maintains track continuity by adjusting segment connections

- Integrated merge into GetMovementAnalysis after movement analysis

### Key Features

- Configurable per-location merge radius

- Preserves track continuity

- Smart jitter detection (only removes movements with both endpoints within radius)

- Backward compatible (default behavior for locations without override)
  
```

后面改进了这个方案，从数据库中查询后的数据，直接进行处理，重写成目标目的地点，这样上层都会很简单。
